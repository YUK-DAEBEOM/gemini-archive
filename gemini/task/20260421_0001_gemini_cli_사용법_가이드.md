# Gemini CLI 효과적인 사용법 가이드

Gemini CLI는 터미널 환경에서 AI 모델의 강력한 성능을 활용하여 코드 이해, 작업 자동화 및 워크플로우 구축을 돕는 도구입니다. 아래는 Gemini CLI를 100% 활용하기 위한 핵심 기능과 권장 사례입니다.

---

## 1. 핵심 기능 (Core Features)

*   **대화형 및 단발성 쿼리**: 
    터미널에서 `gemini`를 입력해 대화형 프롬프트를 시작하거나, `gemini -p "내용"` 형식으로 한 번의 명령을 수행할 수 있습니다.
*   **세션 관리**: 
    모든 대화는 세션으로 저장됩니다. 이전 대화를 이어서 하려면 `gemini -r latest`를 사용하고, `--list-sessions`로 과거 기록을 확인할 수 있습니다.
*   **프로젝트 컨텍스트 (GEMINI.md)**: 
    프로젝트에 `GEMINI.md` 파일을 생성하여 코딩 규칙이나 지침을 작성해두면, 모델이 이를 자동으로 읽어 컨텍스트로 활용합니다.
*   **플랜 모드 (Plan Mode)**: 
    복잡한 작업을 시작하기 전에 모델이 어떻게 코드를 수정할지 계획(Plan)을 세우고, 사용자가 이를 검토(수정)한 뒤 실행하게 함으로써 안전성을 높입니다.
*   **확장 기능**: 
    Agent Skills나 Model Context Protocol(MCP)을 사용하여 GitHub, 데이터베이스 등 외부 도구와 연동할 수 있습니다.

---

## 2. 권장 사례 (Best Practices)

### 2-1. 불필요한 파일 무시하기 (`.geminiignore`)
`.gitignore`와 유사하게 `.geminiignore` 파일을 사용하여 `node_modules`, 바이너리 파일, 대용량 로그 파일 등이 모델에게 노출되지 않도록 설정하세요. 이는 모델의 응답 속도와 정확도를 크게 높여줍니다.

### 2-2. 반복적인 요청은 `GEMINI.md`에 작성하기
"항상 TypeScript로 코드를 작성해줘", "주석은 한글로 달아줘"와 같이 매번 반복해야 하는 규칙이 있다면 `GEMINI.md`에 명시하세요. 전역(`~/.gemini/GEMINI.md`)에 설정하거나 프로젝트 폴더별로 나누어 관리할 수 있습니다.

### 2-3. 플랜 모드에서 에디터 적극 활용하기
플랜 모드에서 모델이 제시한 계획을 검토할 때, 터미널에서 `Ctrl+X`를 눌러 VS Code나 Vim 같은 외부 에디터로 계획서를 열 수 있습니다. 텍스트로 수정 방향을 지시하는 것보다 훨씬 빠르고 정확합니다.

---

## 3. 효과적인 사용 팁 및 단축키

### 단축키 마스터하기
*   `Shift + Tab`: 승인 모드(Default -> Auto-Edit -> Plan)를 빠르게 전환합니다.
*   `Ctrl + R`: 과거 프롬프트 기록을 검색합니다.
*   `F12`: 디버그 콘솔을 열어 도구 호출 내역이나 에러 상세 정보를 확인합니다.

### 터미널 도구와 결합하기 (파이프 연산자)
다른 터미널 명령어의 결과를 Gemini로 넘겨 바로 분석할 수 있습니다.
```bash
# 로그 파일 에러 분석 예시
cat error.log | gemini "이 로그의 에러 원인이 뭐야?"
```

### 점진적으로 자동화 적용하기
처음에는 `default` 모드에서 모델이 실행하는 쉘 명령어와 파일 수정을 하나씩 확인(Confirm)하며 동작 방식을 파악하세요. 이후 신뢰가 쌓이면 `auto_edit`(자동 편집) 모드를 활용하여 반복 작업을 빠르게 처리하는 것을 권장합니다.
