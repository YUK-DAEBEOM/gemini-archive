# Gemini CLI 스킬 및 확장 프로그램 가이드

Gemini CLI의 **Agent Skills(에이전트 스킬)**와 **Extensions(확장 프로그램)**는 CLI의 기능을 확장하고 전문화된 능력을 부여하는 핵심 요소입니다. 

## 1. Agent Skills (에이전트 스킬)

Agent Skills는 특정 작업(예: 코드 리뷰, 배포, 마이그레이션)에 최적화된 **전문 지식, 절차적 워크플로우, 관련 리소스**를 하나로 묶어둔 패키지입니다.

*   **특징**:
    *   **온디맨드 (On-demand)**: 항상 켜져 있는 것이 아니라, 대화 중 특정 스킬이 필요하다고 판단될 때만 모델이 불러옵니다. (토큰 낭비 방지)
    *   **절차 가이드**: 복잡한 작업을 단계별로 수행할 수 있도록 표준화된 가이드라인(`SKILL.md`)을 제공합니다.
*   **사용 방법**:
    *   `/skills list` 명령어로 현재 사용 가능한 스킬 목록을 확인할 수 있습니다.
    *   인터넷이나 로컬 경로에서 설치하려면: `gemini skills install <경로/URL>`
*   **만드는 방법**:
    *   **자동**: 채팅 창에 "새로운 스킬을 만들어줘"라고 요청하면 내장된 `skill-creator`가 뼈대를 생성해 줍니다.
    *   **수동**: `SKILL.md` 파일을 생성하고 맨 위에 YAML 형식으로 스킬의 이름과 설명을 적은 뒤, 아래에 구체적인 지침을 마크다운으로 작성합니다.

## 2. Extensions (확장 프로그램)

Extensions는 새로운 명령어, 모델이 사용할 수 있는 도구(Tools), 공통 컨텍스트 등을 하나로 묶어 **배포 및 공유하기 쉽게 만든 패키지**입니다.

*   **주요 기능**:
    *   **MCP 서버**: 모델이 데이터베이스를 조회하거나 외부 API를 호출할 수 있도록 새로운 도구(Tools)를 장착합니다.
    *   **커스텀 명령어**: 자주 쓰는 긴 프롬프트를 `/내명령어` 처럼 단축어로 만들 수 있습니다.
    *   **Hooks**: CLI가 실행될 때, 혹은 응답하기 전에 특정 스크립트를 가로채서 실행할 수 있습니다.
*   **사용 방법**:
    *   설치: `gemini extensions install <GitHub-URL>`
    *   목록 확인: `/extensions list`
*   **만드는 방법**:
    1.  `gemini extensions new <이름> <템플릿>` 명령어로 기본 프로젝트 템플릿을 생성합니다. (예: `mcp-server` 템플릿)
    2.  생성된 `gemini-extension.json` 파일에 설정 정보를 기입합니다.
    3.  `gemini extensions link .` 명령어로 로컬에 연결하여 개발 및 테스트를 진행할 수 있습니다.

## 📝 요약 비교

| 구분 | Agent Skills | Extensions |
| :--- | :--- | :--- |
| **핵심 목적** | 특정 작업의 가이드라인 및 전문 지식 제공 | CLI 자체의 기능 확장 및 도구 추가 |
| **활성화 시점** | 모델이 필요하다고 판단할 때 호출 | CLI 실행 시 항상 로드 |
| **주요 구성 요소** | `SKILL.md`, 스크립트 | `gemini-extension.json`, MCP 서버 |
| **추천 사용처** | 코드 리뷰봇, 컨벤션 가이드, 문제 해결 절차 | 신규 API 연동, 슬래시(`/`) 명령어 추가 |
