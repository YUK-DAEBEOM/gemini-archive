# Gemini CLI 설치 방법

Gemini CLI를 설치하는 방법은 여러 가지가 있으며, 사용자의 환경에 맞춰 선택할 수 있습니다. 권장되는 설치 및 실행 방법은 다음과 같습니다.

## 1. 주요 설치 방법

가장 일반적으로 권장되는 방법은 **npm**을 이용한 글로벌 설치입니다.

*   **npm (모든 OS)**:
    ```bash
    npm install -g @google/gemini-cli
    ```
*   **Homebrew (macOS/Linux)**:
    ```bash
    brew install gemini-cli
    ```
*   **MacPorts (macOS)**:
    ```bash
    sudo port install gemini-cli
    ```
*   **Anaconda (제한된 환경)**:
    ```bash
    conda create -y -n gemini_env -c conda-forge nodejs
    conda activate gemini_env
    npm install -g @google/gemini-cli
    ```

## 2. 설치 없이 바로 실행하기 (npx)

설치 과정을 생략하고 바로 실행해보고 싶다면 `npx`를 사용할 수 있습니다.
```bash
npx @google/gemini-cli
```

## 3. 특정 릴리스 채널 설치

Gemini CLI는 세 가지 릴리스 채널을 제공합니다. 기본적으로는 `latest`(안정 버전)가 설치됩니다.
*   **안정 버전 (Stable)**: `npm install -g @google/gemini-cli`
*   **미리보기 버전 (Preview)**: `npm install -g @google/gemini-cli@preview`
*   **매일 업데이트 버전 (Nightly)**: `npm install -g @google/gemini-cli@nightly`

## 4. 권장 시스템 사양 및 요구사항

*   **OS**: macOS 15+, Windows 11 24H2+, Ubuntu 20.04+
*   **Runtime**: Node.js 20.0.0 이상
*   **인터넷 연결**: 필수
*   **클라우드 환경**: Gemini CLI는 [Cloud Shell](https://docs.cloud.google.com/shell/docs) 및 [Cloud Workstations](https://cloud.google.com/workstations)에 기본적으로 사전 설치되어 있습니다.

설치가 완료되면 터미널에서 `gemini` 명령어를 입력하여 실행할 수 있습니다.
