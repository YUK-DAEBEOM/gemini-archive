# Gemini CLI 지원 스킬 및 내장 에이전트 리스트

Gemini CLI는 기본적으로 여러 내장 서브에이전트(Subagents)를 제공하며, 필요에 따라 다양한 스킬(Skills)과 확장 프로그램(Extensions)을 추가할 수 있습니다.

## 1. 내장 서브에이전트 (Built-in Subagents)

서브에이전트는 특정 분야에 특화된 전문가 역할을 수행합니다. 메인 에이전트가 작업의 성격을 파악하여 자동으로 호출하거나, 사용자가 멘션하여 직접 호출할 수도 있습니다.

*   **`codebase_investigator`**:
    *   **역할**: 코드베이스 분석, 리버스 엔지니어링, 복잡한 의존성 파악
    *   **사용처**: 버그의 근본 원인을 찾거나, 대규모 리팩토링 전 시스템 구조를 파악할 때 사용됩니다.
*   **`cli_help`**:
    *   **역할**: Gemini CLI 자체 기능, 명령어, 설정에 대한 전문 지식 제공
    *   **사용처**: CLI 사용법이 궁금하거나, 새로운 스킬/확장 프로그램을 만들 때 가이드가 필요할 때 사용됩니다.
*   **`generalist` (Generalist Agent)**:
    *   **역할**: 광범위한 도구 접근 권한을 가진 범용 에이전트
    *   **사용처**: 메인 세션의 기록을 깔끔하게 유지하면서, 여러 파일에 걸친 반복적인 리팩토링이나 대량의 출력이 예상되는 쉘 명령어를 실행할 때 작업을 위임합니다.
*   **`browser_agent` (실험적 기능)**:
    *   **역할**: 웹 브라우저 자동화
    *   **사용처**: 특정 웹 페이지 네비게이션, 폼 입력, 웹 기반 정보 추출 등을 수행합니다.

## 2. 에이전트 스킬 (Agent Skills) 예시

스킬은 특정 작업이 필요할 때만 컨텍스트에 로드되는 온디맨드(On-demand) 워크플로우입니다. 현재 사용 가능한 스킬 목록은 터미널에서 `/skills list`로 확인할 수 있습니다.

*   **`skill-creator`**:
    *   **내장 스킬**: 사용자가 새로운 스킬을 만들고자 할 때 활성화되어, 스킬 제작에 필요한 `SKILL.md` 구조와 템플릿을 자동으로 작성해 주는 가이드 스킬입니다.
*   **커뮤니티/사용자 정의 스킬 (예시)**:
    *   **`api-auditor`**: API 엔드포인트 보안 감사 및 테스트 자동화 스킬 (튜토리얼 예시)
    *   이 외에도 팀 내부의 코드 리뷰 가이드, 클라우드 인프라 배포 절차, 특정 프레임워크 마이그레이션 가이드 등을 직접 스킬로 만들어 사용할 수 있습니다.

## 3. 확장 프로그램 (Extensions) 생태계

확장 프로그램은 터미널에 상주하며, 새로운 도구(Tools)나 명령어(Slash commands)를 추가해 줍니다. 터미널에서 `/extensions list`로 현재 설치된 항목을 볼 수 있습니다.

*   **기본 제공 (Built-in)**: 모델이 로컬 환경과 상호작용하기 위한 파일 읽기/쓰기, 쉘 명령어 실행 도구 등이 내장된 확장 형태로 제공됩니다.
*   **공식 갤러리**: 다양한 MCP(Model Context Protocol) 서버와 연동하여 데이터베이스 쿼리, GitHub 이슈 관리 등을 수행할 수 있는 확장 프로그램들을 [공식 확장 갤러리](https://geminicli.com/extensions/browse/)에서 찾아 설치할 수 있습니다.
