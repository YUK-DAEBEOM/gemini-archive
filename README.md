# 🚀 Gemini CLI 상세 사용 가이드

이 문서는 Gemini CLI의 핵심 기능, 명령어, 그리고 이 프로젝트에서 사용하는 **bkit(PDCA)** 워크플로우를 효과적으로 활용하기 위한 가이드입니다.

---

## 1. 🌟 핵심 기능
Gemini CLI는 단순한 채팅 인터페이스를 넘어 로컬 코드베이스를 이해하고 실행하는 강력한 개발 보조 도구입니다.

- **코드베이스 인식:** 프로젝트 구조와 파일 내용을 실시간으로 파악합니다.
- **쉘 연동:** 터미널 명령어를 직접 실행하거나 파일 시스템을 제어합니다.
- **서브 에이전트:** 코드 분석(`codebase_investigator`), CLI 도움말(`cli_help`) 등 전문 에이전트에게 작업을 위임합니다.
- **메모리 기능:** 프로젝트의 규칙이나 중요한 결정 사항을 `GEMINI.md`에 저장하고 기억합니다.

---

## 2. 🧠 GEMINI.md (프로젝트 핵심 메모리) 관리 가이드
GEMINI.md는 단순한 메모장이나 로그 파일이 아니다. 해당 프로젝트 디렉터리 내에서 Gemini CLI가 강제로 준수해야 하는 시스템 프롬프트(System Prompt)의 확장판으로 동작한다.
- 예시
```
# 프로젝트 제약사항 및 컨벤션
- 프레임워크: FastAPI, Python 3.10+
- 동시성 제어: 멀티프로세싱 구현 시 반드시 Semaphore를 사용하여 자원 경합을 제어할 것.
- DB 접근: ORM 객체 매핑을 최소화하고 복잡한 조회는 Raw SQL을 최우선으로 작성할 것.
- 언어 설정: 주석과 커밋 메시지는 모두 한국어로 작성할 것.
```

## 3. 🛠️ 주요 명령어 (Commands)

명령어는 슬래시(`/`), 골뱅이(`@`), 느낌표(`!`)로 시작합니다.

### 시스템 및 설정
- `/help`: 사용 가능한 모든 명령어 목록과 도움말을 표시합니다.
- `/settings`: CLI 설정 에디터를 엽니다.
- `/model`: 사용할 AI 모델을 확인하거나 변경합니다.

### 컨텍스트 및 세션 관리
- `/resume` (또는 `/chat`): **세션 브라우저**를 열어 이전 대화 목록을 확인하고 원하는 시점으로 전환합니다.
- `/clear`: 현재 대화 내용(컨텍스트)은 유지하되, 터미널 **화면만 깨끗하게 정리**합니다. (단축키: Ctrl+L)
- `/rewind`: 대화 상태를 이전 단계로 되돌립니다. (실행 취소와 유사, 단축키: Esc 두 번)
- `/compress`: 대화 맥락을 요약하여 토큰 사용량을 최적화합니다.
- `/plan`: **플랜 모드**로 전환하여 안전하게 변경 사항을 설계합니다.

### 파일 및 쉘 조작
- `@<경로>`: 특정 파일이나 디렉터리 내용을 대화 컨텍스트에 포함합니다. (예: `@src/main.py 이 파일 분석해줘`)
- `!<명령어>`: 쉘 명령어를 직접 실행합니다. (예: `!git status`, `!npm run test`)
- `/restore`: 도구 실행으로 변경된 파일을 이전 상태로 복구합니다.

## 4. 🔄 bkit (PDCA) 워크플로우

이 프로젝트는 **PDCA(Plan-Design-Do-Check-Act)** 방법론을 따릅니다. 새로운 기능을 추가하거나 수정할 때 다음 명령어를 순서대로 사용하세요.

| 단계 | 명령어 | 용도 |
| :--- | :--- | :--- |
| **Plan** | `/pdca plan [기능명]` | 작업의 목적과 범위를 정의하고 계획서를 작성합니다. |
| **Design** | `/pdca design [기능명]` | 구체적인 구현 방식, 아키텍처, 테스트 전략을 설계합니다. |
| **Do** | `/pdca do [기능명]` | 실제 코드를 작성하고 기능을 구현합니다. |
| **Check** | `/pdca analyze [기능명]` | 구현된 결과물이 설계와 일치하는지 갭 분석을 수행합니다. |
| **Act** | `/pdca iterate [기능명]` | 분석 결과(90% 미만)를 바탕으로 코드를 개선합니다. |
| **Report** | `/pdca report [기능명]` | 작업 완료 보고서를 생성하고 종료합니다. |

---

## 5. 🤖 서브 에이전트 (Sub-agents) 활용

복잡한 작업은 전문 에이전트에게 직접 요청할 수 있습니다.

- **자동 위임:** 질문의 의도에 따라 메인 에이전트가 적절한 전문가를 자동으로 호출합니다.
- **직접 호출:** `@에이전트명 [요청사항]` 형식을 사용합니다.
  - `@codebase_investigator`: 프로젝트의 전체 구조나 복잡한 의존성을 분석할 때 사용합니다.
  - `@cli_help`: Gemini CLI 자체의 기능이나 설정이 궁금할 때 사용합니다.
  - `@generalist`: 대량의 파일 수정이나 반복적인 작업이 필요할 때 사용합니다.

## 6. 💡 효율적인 사용 팁

1. **컨텍스트 최적화:** 너무 많은 파일을 한꺼번에 읽기보다는 `@`를 사용하여 필요한 파일만 정확히 지정하세요.
2. **설계 우선:** 큰 작업을 시작하기 전에는 항상 `/pdca plan` 또는 `/plan` 모드를 사용하여 구조를 먼저 잡으세요.
3. **메모리 활용:** 프로젝트 고유의 규칙(예: 네이밍 컨벤션)이 있다면 `/memory add`를 통해 CLI가 기억하게 하세요.
4. **쉘 연동:** 테스트 실행(`!pytest`)이나 린트 체크(`!ruff check .`)를 대화 중에 바로 수행하여 품질을 검증하세요.

### 6.1. 🔍 효과적인 코드 분석 요청 팁

AI를 통해 더 정확하고 깊이 있는 분석 결과를 얻기 위한 전략입니다.

1. **🎯 컨텍스트(Context) 범위를 명확히 지정하세요 (`@` 활용)**
   - 단순히 질문하기보다 분석에 필요한 파일을 정확히 지정하세요.
   - 예: `@src/service/agents/orchestrator.py 이 에이전트의 주요 로직을 설명해줘.`

2. **🕵️ 분석의 '목적'을 구체적으로 명시하세요**
   - 버그 탐색, 성능 최적화, 코드 가독성 개선 등 원하는 목표를 알려주세요.
   - 예: "이 로직에서 발생할 수 있는 레이스 컨디션이나 예외 상황이 있을까?"

3. **🤖 적절한 서브 에이전트를 활용하세요**
   - 프로젝트 전체 구조나 복잡한 의존성 분석은 `@codebase_investigator`에게 맡기세요.
   - 예: `@codebase_investigator 이 프로젝트의 에이전트 간 데이터 흐름을 시퀀스 다이어그램으로 그려줘.`

4. **🪜 단계적 사고(Chain of Thought)를 유도하세요**
   - AI가 논리적으로 생각을 정리할 수 있게 "단계별(Step-by-step)"로 분석을 요청하세요.
   - 예: "이 함수가 호출될 때 변수가 어떻게 변화하는지 단계별로 설명해줘."

5. **📏 프로젝트 컨벤션과 연결하세요**
   - 프로젝트 고유의 규칙이나 `README.md`의 설계 원칙을 바탕으로 분석을 요청하세요.
   - 예: "우리 프로젝트의 네이밍 컨벤션에 비추어 볼 때 리팩토링할 부분이 있어?"

## 7. 📚 세션 및 이력 관리

Gemini CLI는 프로젝트 단위로 대화 세션을 자동 저장하고 관리합니다.

1. **새로운 세션 시작 및 전환**
   - **새로운 세션:** 터미널에서 `gemini`를 입력하면 새로운 컨텍스트로 대화가 시작됩니다.
   - **이력 확인:** 대화 중 `/resume`을 입력하면 세션 브라우저가 열리며, 과거 대화 목록을 검색하고 선택할 수 있습니다.
   - **특정 세션 재개:** 터미널에서 `gemini --resume [인덱스]` 명령어로 바로 시작할 수 있습니다.

2. **`/resume` vs `/clear` 차이점**
   - **`/resume` (맥락 전환):** 과거의 특정 시점으로 돌아가 대화 내용과 실행 이력을 모두 복구합니다. (칠판을 갈아끼우는 것)
   - **`/clear` (화면 정리):** AI의 기억(컨텍스트)은 그대로 유지하면서 터미널 화면의 가시적인 텍스트만 지웁니다. (적힌 내용은 기억하되 칠판만 닦는 것)

3. **수동 체크포인트 활용 (`/resume save`)**
   - 중요한 결정이나 구현 직전에 `/resume save [태그명]`으로 시점을 저장해 두세요.
   - 나중에 `/resume resume [태그명]`으로 해당 시점부터 다시 시작하거나 다른 방향으로 구현을 시도(Fork)할 수 있어 매우 유용합니다.

## 8. 📊 사용량 및 요금 확인

Gemini CLI 내에서 현재 세션의 토큰 소모량과 할당량(Quota)을 확인하는 방법입니다.

1. **내부 통계 명령어**
   - `/stats model`: 현재 모델의 토큰 사용량과 적용된 할당량 정보를 실시간으로 보여줍니다.
   - `/stats session`: 현재 세션의 총 지속 시간, 도구 호출 횟수, 성능 지표를 요약합니다.
   - `/stats tools`: 각 도구(파일 읽기, 쉘 실행 등)가 얼마나 사용되었는지 확인합니다.

2. **외부 대시보드 (상세 요금 및 잔액 확인)**
   - **Gemini API Key 사용자:** [Google AI Studio 대시보드](https://aistudio.google.com/app/plan_billing)에서 상세 사용량과 플랜 정보를 확인할 수 있습니다.
   - **Vertex AI (Enterprise) 사용자:** [Google Cloud Console 할당량 페이지](https://console.cloud.google.com/iam-admin/quotas)에서 프로젝트별 한도를 확인하세요.

3. **토큰 절약 팁**
   - 대화가 길어지면 과거 메시지가 모두 토큰으로 소모됩니다. `/compress` 명령어를 사용하여 대화 맥락을 요약하면 이후 요청 시 소모되는 토큰을 크게 줄일 수 있습니다.
